# Code of Conduct

## Our standard

Contributors are expected to be respectful, direct, and constructive.

Examples of expected behavior:

- giving actionable technical feedback
- respecting different experience levels
- focusing discussion on the code and product behavior
- assuming good intent while still being precise about problems

Unacceptable behavior includes:

- harassment or discrimination
- personal attacks
- doxxing or sharing private account/session information
- disruptive or bad-faith participation

## Enforcement

Project maintainers may remove comments, issues, pull requests, or participants that violate this code of conduct.
